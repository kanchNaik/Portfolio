    Med-VQA: Multimodal Medical Visual Question Answering

    A research project exploring Medical Visual Question Answering (VQA) using a LoRA fine-tuned LLaVA-Med model and OpenAI model integrated with multimodal Retrieval-Augmented Generation (RAG) and advanced prompting strategies (Few-Shot, Chain-of-Thought, Tree-of-Thought) to enhance accuracy and explainability. The system retrieves relevant visual (X-rays/MRIs) and textual (PubMedQA, SLAKE, PMC-VQA) data to generate informed answers with strong visual-language reasoning capabilities.


    • Overview: System that answers medical questions from X-rays/MRIs using visual + textual retrieval.
    • Techniques: Prompting (Few-Shot, CoT, ToT), RAG with PubMedQA, SLAKE, PMC-VQA
    • Fine-tuning: Lightweight LoRA-tuned LLaVA-Med
    • Performance: CoT/ToT Recall: 77%, LoRA Accuracy: 54.27%, RAG Recall: 71.83%
    • Technologies: Python, PyTorch, FAISS, CLIP, MiniLM, GPT-4 API, LLaVA-Med
    • Features: Explainability, Visual + Text reasoning, Few-shot & zero-shot inference

    • Goal: Simulate personality-driven Q&A using real-world interviews, books, and podcasts

    • Results: Avg. human-rated personality alignment scores: Elon (4.7), Freeman (4.46), Attenborough (4.1)

    Agentic AI: Multi-Agent Code Generation System

    A modular, agent-based AI architecture designed to convert natural language requests into validated frontend + backend code using API documentation and retrieval-augmented generation (RAG). Built to simulate intelligent coordination among agents and support iterative feedback loops.


    • Key Agents: Instruction Parser, Retriever (RAG), Code Generator, Code Validator, Feedback Handler, Conversation State Manager
    • RAG Setup: Uses BGE-small-en for embedding API docs (from CSV), stored in FAISS + metadata dictionary
    • Code Generator: Combines user intent, subtasks, and retrieved API docs to generate full-stack code using GPT-4 (or Mistral/LLaMA)
    • Validation Layer: Syntax check, linting, API match, and security screening before user delivery
    • Feedback Loop: User rejection triggers dynamic re-prompting using stored session history, enhancing regeneration accuracy
    • Tech Stack: Python, FAISS, Sentence-BERT, BeautifulSoup, Playwright, GPT-4, Mistral, Pydantic
